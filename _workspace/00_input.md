# 꼬리질문 로직 전면 개편

## 요청
Web Speech API 의존 제거 → Whisper STT + LLM 교정 완료 후 Haiku 모델로 꼬리질문 판단+생성

## 요구사항
1. 5초 미만 답변 → 꼬리질문 X (회피/테스트)
2. 5초 이상 → STT + 교정 완료 대기 → 교정된 텍스트 기반 LLM 분석
3. 판단 기준 (3가지 모두 충족 시 패스):
   - 질문 의도에 부합하는 답변
   - 구체적 사례/경험 1개 이상
   - 본인의 역할/행동 명시
4. 하나라도 빠지면 → 해당 유형에 맞는 꼬리질문 생성
5. 대기 중 "면접관이 답변을 검토 중입니다" + 프로그레스 바
6. 하드모드에서도 동일 적용 (꼬리질문 나오면 타이핑+카운트다운+자동녹화)
7. 꼬리질문 판단+생성은 Haiku 모델 사용 (비용 절감, 속도 향상)

## 제약
- 자기소개 질문은 꼬리질문 스킵
- 5질문 중 최대 2~3개만 (LLM 프롬프트로 제어)
- STT 실패 시 → 녹화 시간 기준 폴백
