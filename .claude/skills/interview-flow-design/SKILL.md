# Interview Flow Design — 면접 플로우 설계 가이드

interview-designer 에이전트가 면접 플로우를 설계하거나 수정할 때 활용하는 패턴 레퍼런스.

## 대상 에이전트

`interview-designer` — 이 스킬의 패턴을 면접 설계에 직접 적용한다.

## 면접 플로우 아키텍처

```
[설정] → [면접 시작] → [질문 제시] → [답변 녹화] → [꼬리질문 판단] → [다음 질문 또는 종료] → [분석] → [리포트]
```

### 상태 전이

| 현재 상태 | 이벤트 | 다음 상태 | 조건 |
|----------|--------|----------|------|
| setup | 면접 시작 클릭 | recording | 쿼타 확인 통과 |
| recording | 질문 제시 | answering | — |
| answering | 답변 완료 | evaluating | 녹화 중지 |
| evaluating | 꼬리질문 필요 | answering | 녹화 시간 기준 충족 |
| evaluating | 꼬리질문 불필요 | next_question | 시간 미충족 또는 마지막 질문 |
| next_question | 다음 질문 있음 | recording | — |
| next_question | 모든 질문 완료 | analyzing | — |
| analyzing | 분석 완료 | report | — |

## 질문 선택 알고리즘

### 가중 랜덤 선택
- **major 항목**: 60% 확률로 선택 — 이력서/포폴의 핵심 경험
- **minor 항목**: 40% 확률로 선택 — 보조 경험, 기술 스택
- 중복 방지: 선택된 항목은 풀에서 제거

### 2단계 질문 생성

1단계 — **주제 추출**: 이력서/포폴에서 면접 가능한 주제를 추출 (LLM)
2단계 — **질문 생성**: 추출된 주제별로 구체적 면접 질문 생성 (LLM)

이 분리의 이유: 한 번에 질문을 생성하면 특정 주제에 편향되는 문제가 발생한다. 주제를 먼저 균등하게 추출한 후 각 주제별로 질문을 생성하여 편향을 방지한다.

### 카테고리별 질문 풀
| 카테고리 | 예시 |
|---------|------|
| 기술 질문 | 프로젝트 경험, 기술 스택 깊이, 설계 판단 |
| 상황 질문 | 문제 해결, 팀 협업, 갈등 관리 |
| 역량 질문 | 학습 방법, 성장 경험, 목표 |

## 꼬리질문 전략

### 녹화 시간 기반 판단
- 답변 녹화 시간이 기준값 이상 → 꼬리질문 가능
- 답변이 너무 짧으면 → 꼬리질문 대신 다음 질문으로
- 꼬리질문 최대 횟수 제한 (질문당 1~2회)

### 꼬리질문 생성 패턴
| 패턴 | 트리거 | 예시 |
|------|--------|------|
| 구체화 | 추상적 답변 | "좀 더 구체적으로 설명해주시겠어요?" |
| 심화 | 충분한 답변 | "그 경험에서 가장 어려웠던 점은?" |
| 결과 확인 | 과정만 설명 | "최종 결과는 어떠했나요?" |
| 기술 검증 | 기술 언급 | "해당 기술을 선택한 이유는?" |

## 평가 루브릭 설계

### 텍스트 분석 기준
| 항목 | 설명 | 가중치 |
|------|------|--------|
| 논리성 | 답변의 구조와 논리적 흐름 | 30% |
| 구체성 | 실제 경험과 수치의 구체성 | 25% |
| 전문성 | 기술적 깊이와 정확성 | 25% |
| 커뮤니케이션 | 표현의 명확성과 자연스러움 | 20% |

### 비전 분석 기준
| 항목 | 설명 |
|------|------|
| 시선 | 카메라 응시 비율 |
| 자세 | 상체 안정성, 바른 자세 유지 |
| 표정 | 자연스러운 표정 변화, 미소 |
| 제스처 | 적절한 손 제스처 사용 |
